# Pantry Economy Phase 3 — Processing Research Gates Implementation Plan

> **For agentic workers:** REQUIRED SUB-SKILL: Use superpowers:subagent-driven-development (recommended) or superpowers:executing-plans to implement this plan task-by-task. Steps use checkbox (`- [ ]`) syntax for tracking.

**Goal:** Add the research layer that gates the two processing buildings — a **kickoff research** that puts each building in the Carpenter's stock, **expansion researches** that unlock the final (non-purchasable) slots, and **per-building fuel-efficiency researches** that shave per-slot wood burn.

**Architecture:** Pure, testable helpers go in `ProcessingMath` (EconomyCore): one computes the effective purchasable slot cap from the number of unlocked expansions; one applies a clamped efficiency reduction to per-slot burn. The two building managers (`CanneryManager`, `SmokehouseManager`) become thin consumers — they read `ResearchManager.IsFeatureUnlocked(flag)` for the cap and `ResearchManager.GetBonus(statKey)` for the burn reduction, and re-fire their own `OnChanged` when a flag unlocks so open popups refresh. `CarpenterPopupUITK` hides each build row until its kickoff flag is set. All seven research entries are authored in `ResearchCatalogGenerator` (Equipment branch) and emitted to `Resources/Research/` by the existing editor menu.

**Tech Stack:** Unity 6000.3 (6000.3.9f1), C#, UI Toolkit, NUnit EditMode tests, Unity MCP (gladekit / UnityMCP) for scene + catalog-regen work.

**Spec:** `docs/superpowers/specs/2026-07-07-pantry-economy-design.md` — §5 (progression & gating), §5a (slot pricing curve), §6 (burn-efficiency lever), and the **"Phase 3 implementation decisions (2026-07-09)"** addendum (the binding decisions for this plan).

```csharp
    /// <summary>
    /// Purchasable slot cap after research expansions (spec §5, Phase 3). Each unlocked expansion adds
    /// `slotsPerExpansion` slots on top of the coin-purchasable base, clamped to the building's hard
    /// total. Negative inputs are treated as zero.
    /// </summary>
    public static int EffectiveSlotCap(int basePurchasable, int totalMax, int expansionsUnlocked, int slotsPerExpansion)
        => Mathf.Min(totalMax, basePurchasable + Mathf.Max(0, expansionsUnlocked) * Mathf.Max(0, slotsPerExpansion));

    /// <summary>
    /// Per-slot wood burn after a fuel-efficiency research bonus (spec §6). `efficiencyBonus` is a
    /// cumulative fraction (as returned by ResearchManager.GetBonus); `maxReduction` clamps total
    /// savings so demand never hits zero and the "keep chopping" loop survives. Only per-slot burn is
    /// reduced — base (empty-fire waste) burn is untouched.
    /// </summary>
    public static float EffectiveBurnPerSlot(float basePerSlotBurn, float efficiencyBonus, float maxReduction)
        => basePerSlotBurn * (1f - Mathf.Clamp(efficiencyBonus, 0f, Mathf.Clamp01(maxReduction)));
```
